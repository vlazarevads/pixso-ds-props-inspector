# Pixso Design System Props Inspector

Плагин для Pixso, который помогает анализировать и проверять пропсы компонентов на соответствие правилам дизайн-системы.

Инструмент позволяет быстро увидеть различия между именами пропов в Pixso, дизайне и коде, а также выявить ошибки в нейминге.

---

## Возможности

* Просмотр всех пропов выбранного компонента
* Сравнение названий пропов:

  * Pixso
  * Design
  * Code
* Проверка соответствия правилам нейминга
* Подсветка проблемных пропов

### Работа с данными

* Копирование результата анализа в JSON

### Генерация документации

* Автоматическая генерация документации компонентов
* Создание блоков документации для каждого пропа
* Генерация демонстраций вариантов пропов
* Экран загрузки во время генерации
* Индикатор прогресса генерации `Проп X из Y`
* Генерация полной документации: три фрейма — Props, How to use, Dark mode
* Автоматическое позиционирование фреймов на холсте
* Фокусировка камеры на сгенерированной документации
* Предупреждение о перезаписи при повторной генерации

### How to use

* Экран контекста перед генерацией — необязательное поле для описания кейсов, которые нужно покрыть
* Генерация раздела How to use через Claude Code CLI (`/generate-how-to-use`)
* Импорт результата в плагин через вставку JSON
* Автоматическое построение навигации из заголовков секций
* Секции с примерами компонента и идеи/антипаттерны в отдельных блоках

---

## Статусы проверки

| Статус         | Описание                    |
| -------------- | --------------------------- |
| **OK**         | проп соответствует правилам |
| **WRONG_CASE** | отличается регистр символов |
| **REVIEW**     | требует проверки            |
| **UNKNOWN**    | проп отсутствует в словаре  |

## Интерфейс плагина

Плагин использует state machine с несколькими экранами:

* **Idle** — плейсхолдер с иконкой и подсказкой, кнопка инспекции
* **Table** — заголовок компонента, цветная статистика пропов, таблица, дропдаун генерации
* **Tech-components** — список вложенных компонентов с чекбоксами для выбора технических
* **Confirm** — предупреждение о перезаписи существующих фреймов
* **How to use prompt** — плейсхолдер с предложением сгенерировать раздел How to use
* **How to use context** — поле для необязательного контекста (кейсы, которые нужно покрыть)
* **How to use import** — инструкция по работе с Claude Code CLI и поле для вставки JSON
* **Loading** — GIF на всю высоту, оверлей с прогрессом, задизейбленный футер

## Генерация документации

Плагин может автоматически создавать структуру документации для компонентов дизайн-системы.

После нажатия **Документация по пропам**:

1. создаётся фрейм `Doc / ComponentName`
2. генерируются блоки для каждого пропа
3. заполняются описания
4. создаются демонстрации вариантов

После нажатия **Полная документация** плагин сначала предлагает выбрать технические компоненты из вложенности, затем создаёт фреймы:

* `Doc / ComponentName` — документация по пропам основного компонента
* `Doc / _TechComponentName` — документация для каждого выбранного технического компонента
* `How to use / ComponentName` — навигация + блоки секций + блоки идей
* `Dark mode / ComponentName` — матрица вариантов компонента (строки × столбцы состояний)

Если фреймы уже существуют, плагин предупреждает о перезаписи и удаляет старые перед созданием новых.

Фреймы располагаются 500px ниже компонента, слева направо с отступом 150px между ними. Камера автоматически фокусируется на всех трёх фреймах.

Во время генерации отображается экран загрузки и прогресс: «Проп 3 из 10»

После завершения полной генерации плагин предлагает сгенерировать раздел How to use через Claude Code CLI.

## Использование

1. Открыть Pixso
2. Запустить плагин
3. Выбрать компонент
4. Нажать **Inspect component**

Плагин покажет таблицу пропов и результаты проверки.

---

## Назначение

Инструмент предназначен для команд, которые используют дизайн-систему и хотят:

* контролировать нейминг пропов
* синхронизировать дизайн и код
* находить ошибки в компонентах
